# Прокси для ИИ-генерации (Cloudflare Worker)

Прячет ключ OpenRouter на сервере: посетители сайта получают ИИ-поздравления,
ничего не вводя. Бесплатный тариф Cloudflare — до 100 000 запросов в день.

## Установка (10 минут, всё бесплатно)

### 1. Создать Worker
1. Зарегистрируйтесь / войдите: **dash.cloudflare.com**
2. Слева: **Workers & Pages** → **Create** → **Create Worker**
3. Имя: `kod-sudby` → **Deploy**
4. Нажмите **Edit code**, удалите всё и вставьте содержимое файла
   [`worker.js`](worker.js) → **Deploy**

### 2. Добавить секретный ключ
1. Вернитесь на страницу воркера → вкладка **Settings** → **Variables and Secrets**
2. **Add** → тип **Secret**, имя `OPENROUTER_KEY`, значение — ваш ключ
   `sk-or-v1-...` с openrouter.ai → **Save and deploy**

### 3. (Необязательно) Честный лимит «3 бесплатно»
Без этого шага лимит считается только в браузере посетителя.
1. **Storage & Databases** → **KV** → **Create namespace**, имя `kod-sudby-limits`
2. У воркера: **Settings** → **Bindings** → **Add** → **KV namespace**,
   имя переменной `LIMITS`, выберите созданный namespace → **Save and deploy**
3. Лимит можно менять переменной `FREE_LIMIT` (по умолчанию 3)

### 4. Подключить сайт
1. Скопируйте адрес воркера — он показан на его странице, вида
   `https://kod-sudby.ВАШ-ЛОГИН.workers.dev`
2. В `index.html` (корень репозитория) найдите строку
   `const PROXY_URL = ''` и впишите адрес:
   `const PROXY_URL = 'https://kod-sudby.ВАШ-ЛОГИН.workers.dev'`
3. Сохраните — GitHub Pages обновится сам через минуту

## Как это работает
- Сайт шлёт воркеру только текст запроса (промпт) — без ключей
- Воркер проверяет лимит по IP (если подключён KV), добавляет секретный ключ
  и обращается к OpenRouter (модель `moonshotai/kimi-k2:free`, можно сменить
  переменной `MODEL`)
- Если лимит исчерпан — сайт показывает окно подписки
- Если воркер недоступен — сайт автоматически откатывается на встроенный
  генератор, посетитель никогда не видит ошибку
